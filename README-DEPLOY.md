# ЧАСТИНА 1 — FIREBASE (база даних)
### Час: ~20 хвилин

---

## Крок 1.1 — Відкрити Firebase Console

1. Відкрийте браузер (Chrome або Firefox)
2. Перейдіть на: **https://console.firebase.google.com**
3. Натисніть **"Sign in"** якщо не ввійшли
4. Увійдіть через свій Google акаунт

Ви побачите сторінку з написом **"Welcome to Firebase"** або список проєктів.

---

## Крок 1.2 — Створити новий проєкт

1. Натисніть велику кнопку **"+ Add project"** (або **"Create a project"**)

2. У полі **"Project name"** введіть:
   ```
   mageqrrest
   ```
   ⚠️ Під полем автоматично з'явиться Project ID — залиште як є.

3. Натисніть **"Continue"**

4. На наступному екрані **"Google Analytics"**:
   - Перемикач можна залишити увімкненим або вимкнути — не важливо
   - Натисніть **"Continue"** (або **"Create project"**)

5. Якщо з'явились Analytics налаштування → виберіть Default Account → **"Create project"**

6. Побачите анімацію "Your new project is ready" → натисніть **"Continue"**

Ви потрапили в дашборд проєкту. Зліва є вертикальне меню.

---

## Крок 1.3 — Створити базу даних Firestore

1. У лівому меню натисніть **"Build"** (розгорнеться підменю)
2. Натисніть **"Firestore Database"**
3. Натисніть синю кнопку **"Create database"**

4. З'явиться вікно налаштувань:
   - Виберіть **"Start in production mode"**
   - Натисніть **"Next"**

5. Вибір локації:
   - Натисніть на випадаючий список
   - Виберіть **"eur3 (Europe)"**
   - Натисніть **"Enable"**

6. Зачекайте 10-30 секунд поки база створюється

Ви побачите порожню базу — це нормально, дані будуть додаватись через сайт.

---

## Крок 1.4 — Налаштувати Authentication (систему входу)

1. У лівому меню → **Build** → **Authentication**
2. Натисніть **"Get started"**

3. Ви бачите список способів входу. Знайдіть **"Email/Password"** і натисніть на нього.

4. У вікні:
   - Перший перемикач **"Email/Password"** → переведіть у **синій (увімкнений)**
   - Другий перемикач **"Email link"** → залиште вимкненим
   - Натисніть **"Save"**

5. Перейдіть на вкладку **"Users"** (зверху сторінки)

6. Натисніть **"Add user"** → заповніть для супер-адміна:
   - Email: `super@mageqrrest.com`
   - Password: придумайте надійний пароль (мінімум 8 символів)
   - **⚠️ ЗАПИШІТЬ ЦЕЙ ПАРОЛЬ — він більше не показується!**
   - Натисніть **"Add user"**

7. Знову **"Add user"** → для адміна демо-закладу:
   - Email: `demo@mageqrrest.com`
   - Password: придумайте і запишіть
   - Натисніть **"Add user"**

---

## Крок 1.5 — Отримати ключі підключення

1. Натисніть на **шестерню ⚙️** у лівому верхньому куті (поруч з "Project Overview")
2. Виберіть **"Project settings"**

3. Прокрутіть сторінку вниз до розділу **"Your apps"**
4. Натисніть на іконку **`</>`** (веб-застосунок)

5. У полі **"App nickname"** введіть: `mageqr-web`
6. ❌ Галочку "Firebase Hosting" НЕ ставте
7. Натисніть **"Register app"**

8. З'явиться блок коду з об'єктом `firebaseConfig`. Він виглядає так:
   ```
   apiKey: "AIzaSyXXXXXXXXXXXXXXXXX"
   authDomain: "mageqrrest.firebaseapp.com"
   projectId: "mageqrrest"
   storageBucket: "mageqrrest.appspot.com"
   messagingSenderId: "123456789"
   appId: "1:123456789:web:abc123"
   ```
   **Зробіть скріншот або скопіюйте всі 6 значень у нотатки.**

9. Натисніть **"Continue to console"**

---

## Крок 1.6 — Вставити ключі у файл firebase-config.js

1. Знайдіть на комп'ютері файл **`firebase-config.js`** (з завантажених файлів)
2. Відкрийте його через Блокнот (Windows) або TextEdit (Mac)
   - Windows: правий клік → "Відкрити за допомогою" → Блокнот
   - Mac: правий клік → "Відкрити за допомогою" → TextEdit

3. Знайдіть цей блок на початку файлу:
   ```
   apiKey:            "ЗАМІНІТЬ_НА_СВІЙ_API_KEY",
   authDomain:        "ВАШ_ПРОЄКТ.firebaseapp.com",
   projectId:         "ВАШ_ПРОЄКТ_ID",
   storageBucket:     "ВАШ_ПРОЄКТ.appspot.com",
   messagingSenderId: "ВАШ_SENDER_ID",
   appId:             "ВАШ_APP_ID"
   ```

4. Замініть кожне значення у лапках на своє зі скріншоту.
   Наприклад рядок `"ЗАМІНІТЬ_НА_СВІЙ_API_KEY"` замініть на `"AIzaSyXXXXXXXXXXX"`

5. Збережіть файл: Ctrl+S (Windows) або Cmd+S (Mac)

---

## Крок 1.7 — Встановити правила безпеки бази даних

1. Поверніться у Firebase Console
2. Зліва → **Firestore Database**
3. Натисніть вкладку **"Rules"** (між "Data" і "Indexes")

4. У полі з правилами — виділіть весь текст (Ctrl+A / Cmd+A) і видаліть його

5. Відкрийте файл **`firestore.rules`** через текстовий редактор
6. Виділіть весь текст (Ctrl+A) → скопіюйте (Ctrl+C)

7. Вставте у поле в Firebase Console (Ctrl+V)

8. Натисніть **"Publish"**

Правила збережено. База тепер захищена від стороннього доступу.

# ЧАСТИНА 2 — GITHUB (хостинг сайту)
### Час: ~15 хвилин

---

## Крок 2.1 — Відкрити GitHub

1. Перейдіть на: **https://github.com**
2. Натисніть **"Sign in"** і увійдіть у свій акаунт

---

## Крок 2.2 — Створити репозиторій

Репозиторій — це папка на GitHub де зберігаються файли вашого сайту.

1. Натисніть кнопку **"+"** у правому верхньому куті сторінки
2. Виберіть **"New repository"**

3. Заповніть форму:

   **Repository name:** (назва папки)
   ```
   mageqr
   ```
   ⚠️ Тільки маленькі літери, без пробілів і спеціальних символів

   **Description:** (необов'язково)
   ```
   Mage QR Rest Platform
   ```

   **Visibility:** оберіть **"Public"**
   ⚠️ Обов'язково Public — інакше GitHub Pages не працює безкоштовно

   **Initialize this repository:**
   ✅ Поставте галочку **"Add a README file"**

   Все інше — залиште за замовчуванням.

4. Натисніть **"Create repository"**

Ви потрапили в порожній репозиторій з одним файлом README.md.

---

## Крок 2.3 — Завантажити всі файли сайту

1. Натисніть кнопку **"Add file"** (вгорі, поруч із зеленою кнопкою Code)
2. Виберіть **"Upload files"**

3. Відкрийте папку з завантаженими файлами на комп'ютері

4. Виберіть ВСІ ці 9 файлів одночасно
   (тримайте Ctrl на Windows або Cmd на Mac щоб вибирати кілька):
   ```
   index.html          ← роутер
   login.html          ← сторінка входу
   admin.html          ← адмін закладу
   superadmin.html     ← панель платформи
   menu.html           ← публічне меню
   anti-scraping.js    ← захист від парсингу
   firebase-config.js  ← ваші ключі Firebase (вже заповнені)
   firestore.rules     ← правила безпеки
   README-DEPLOY.md    ← ця інструкція
   ```
   ⚠️ Файл `venue-admin.html` не потрібен — він вже перейменований в `admin.html`

5. Перетягніть вибрані файли у сіру область на GitHub
   або натисніть **"choose your files"** і виберіть їх

6. Дочекайтесь поки файли завантажаться (бачите список назв — це нормально)

7. Знизу є секція **"Commit changes"**:
   - У першому полі (заголовок) напишіть:
     ```
     Initial deploy — Mage QR Rest
     ```
   - Решту полів залиште як є
   - Натисніть зелену кнопку **"Commit changes"**

8. Ви повернетесь до репозиторію і побачите всі файли у списку ✅

---

## Крок 2.4 — Увімкнути GitHub Pages

1. У верхньому рядку вкладок репозиторію натисніть **"Settings"**
   ⚠️ Це Settings репозиторію, не вашого акаунту!

2. У лівому меню прокрутіть вниз до розділу **"Code and automation"**
3. Натисніть **"Pages"**

4. Знайдіть секцію **"Build and deployment"**:
   - **Source:** у випадаючому списку виберіть **"Deploy from a branch"**
   - **Branch:** у першому списку виберіть **"main"**
   - У другому списку залиште **"/ (root)"**
   - Натисніть **"Save"**

5. Сторінка оновиться. Зачекайте 1-3 хвилини і оновіть сторінку (F5).

6. Зверху з'явиться зелений блок:
   ```
   Your site is live at:
   https://ВАШ_НІКНЕЙМ.github.io/mageqr/
   ```

7. **Скопіюйте і збережіть це посилання** — це адреса вашого сайту!

---

## Крок 2.5 — Додати домен у Firebase

Без цього кроку Firebase відхилятиме спроби входу з вашого сайту.

1. Поверніться у Firebase Console: **https://console.firebase.google.com**
2. Виберіть ваш проєкт **mageqrrest**
3. Зліва → **Build** → **Authentication**
4. Натисніть вкладку **"Settings"** (не "Sign-in method", а Settings)
5. Знайдіть розділ **"Authorized domains"**
6. Натисніть **"Add domain"**
7. Введіть ваш домен GitHub Pages:
   ```
   ВАШ_НІКНЕЙМ.github.io
   ```
   Наприклад: `johnsmith.github.io`
8. Натисніть **"Add"**

---

## Крок 2.6 — Перевірити що сайт працює

1. Відкрийте ваше посилання: `https://ВАШ_НІКНЕЙМ.github.io/mageqr/`

2. Ви повинні побачити темну сторінку з логотипом **✦ Mage QR Rest**
   і анімацією завантаження → вона автоматично переходить на сторінку входу

3. Перевірте вхід (поки що демо-режим):
   - Email: `demo@mageqrrest.com`
   - Пароль: той що задали у Кроці 1.4
   - Натисніть "Увійти" → відкриється адмін-панель ✅

4. Перевірте публічне меню:
   Відкрийте: `https://ВАШ_НІКНЕЙМ.github.io/mageqr/menu.html?v=demo`
   Ви побачите публічне меню ✅

5. Перевірте захист від парсингу:
   Спробуйте правою кнопкою миші → "Переглянути код" на сторінці меню
   → контекстне меню заблоковано ✅

## Крок 3.1 — Оновити login.html для роботи з Firebase

Зараз логін у демо-режимі (перевіряє пароль локально).
Щоб підключити реальну Firebase, потрібно змінити кілька рядків.

1. Відкрийте **`login.html`** у текстовому редакторі

2. Знайдіть (пошук Ctrl+F) цей текст:
   ```
   FIREBASE AUTH (підключити після налаштування Firebase)
   ```

3. Розкоментуйте 3 рядки нижче — видаліть `//` на початку кожного:
   ```javascript
   // const { signInWithEmailAndPassword } = ...
   // const userCred = await signInWithEmailAndPassword(...)
   // window.location.replace('admin.html');
   ```
   Стане:
   ```javascript
   const { signInWithEmailAndPassword } = ...
   const userCred = await signInWithEmailAndPassword(...)
   window.location.replace('admin.html');
   ```

4. Знайдіть рядок:
   ```
   DEMO MODE (видалити після підключення Firebase)
   ```

5. Видаліть весь демо-блок (від цього коментаря до `}` що закриває блок)

6. Збережіть файл

7. Завантажте оновлений файл на GitHub:
   - Відкрийте репозиторій на GitHub
   - Натисніть на `login.html` у списку файлів
   - Натисніть іконку олівця ✏️ праворуч
   - Виділіть весь старий текст → вставте новий
   - Натисніть **"Commit changes"** → **"Commit changes"** ще раз

## Якщо щось пішло не так

**Сайт не відкривається після 5 хвилин:**
→ Settings → Pages → переконайтесь що Source = "Deploy from branch", Branch = "main"
→ Перевірте що репозиторій Public (не Private)

**Бачите 404 помилку:**
→ Переконайтесь що файл `index.html` є в репозиторії (не в підпапці)

**Помилка при вході:**
→ Перевірте що в `firebase-config.js` правильні ключі
→ Перевірте що домен `ВАШ_НІКНЕЙМ.github.io` доданий у Firebase Auth → Settings → Authorized domains

**Файли завантажились але сайт старий:**
→ Зачекайте 2-3 хвилини (GitHub Pages кешує)
→ Оновіть сторінку з Ctrl+Shift+R (жорстке оновлення)
